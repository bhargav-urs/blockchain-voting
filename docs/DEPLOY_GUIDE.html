Install Prerequisites
One-time setup on your Mac — skip anything already installed
A

Check if Git is installed

Open Terminal (press ⌘ + Space, type "Terminal", press Enter) and run:

git --version

If you see something like git version 2.x.x — you're good. If not, go to git-scm.com and download Git for Mac.

B

Check if Node.js is installed

node --version

You need version 18 or higher. If not installed, go to nodejs.org → download the LTS version → install it.

C

Make sure MetaMask is installed

You already have this from local testing. ✅

D

Create a GitHub account (if you don't have one)

Go to github.com → Sign up → it's free. Remember your username — you'll need it.

E

Create a Vercel account (if you don't have one)

Go to vercel.com → Continue with GitHub → Authorise. This links them together so deployment is automatic later.

2
Create Admin Wallet & Get Free MATIC
Your wallet deploys the contract and becomes the permanent admin
A

Create a dedicated admin wallet in MetaMask

Don't use your existing wallet — create a fresh one just for ChainVote:

1. Open MetaMask → click the circle icon at the top right
2. Click "Add account or hardware wallet"
3. Click "Add new Ethereum account"
4. Name it ChainVote Admin → click Create

B

Export and save the private key

⚠️ Never share this key with anyone. Never put it in GitHub. This key controls your admin wallet.

1. In MetaMask, click the three dots (⋮) next to "ChainVote Admin"
2. Click "Account details"
3. Click "Show private key"
4. Enter your MetaMask password
5. Copy the key — it starts with 0x and is 66 characters long
6. Save it in your Notes app or password manager

C

Copy your wallet address

In MetaMask, make sure "ChainVote Admin" is selected. Click the address shown below the account name — it copies it. Save this too. It looks like: 0xf39Fd6...2266

D

Add Polygon Amoy network to MetaMask

1. Open MetaMask → click the network dropdown at the top
2. Click "Add a custom network" (or "Add network manually")
3. Fill in exactly:

FieldValue
Network namePolygon Amoy
New RPC URLhttps://rpc-amoy.polygon.technology/
Chain ID80002
Currency symbolMATIC
Block explorer URLhttps://amoy.polygonscan.com

4. Click Save → MetaMask switches to Polygon Amoy

E

Get free test MATIC from the faucet

This pays the gas fee to deploy the contract. Completely free:

1. Go to faucet.polygon.technology
2. Select Amoy from the network dropdown
3. Paste your ChainVote Admin wallet address
4. Click Submit
5. Wait ~30 seconds

✅ Verify you received MATIC: go to amoy.polygonscan.com → search your wallet address → you should see 0.5 MATIC balance.
ℹ️ If the faucet is down, try: faucets.chain.link or mumbaifaucet.com (select Amoy).
3
Configure Your Project
Set up .env.local with your private key and network settings
A

Open your project folder in Terminal

cd ~/Desktop/blockchain-voting
# or wherever you unzipped the project
B

Create your .env.local file

cp .env.example .env.local

Now open .env.local in any text editor (TextEdit, VS Code, Cursor, etc.) and fill it in exactly like this:

# ── Network ─────────────────────────────────────────────
NEXT_PUBLIC_RPC_URL=https://rpc-amoy.polygon.technology/
NEXT_PUBLIC_CHAIN_ID=80002
NEXT_PUBLIC_CHAIN_HEX=0x13882
NEXT_PUBLIC_CHAIN_NAME=Polygon Amoy
NEXT_PUBLIC_EXPLORER_BASE_URL=https://amoy.polygonscan.com

# ── Contract (deploy script fills this automatically) ────
NEXT_PUBLIC_FACTORY_ADDRESS=
NEXT_PUBLIC_ADMIN_ADDRESS=0xYOUR_ADMIN_WALLET_ADDRESS

# ── Deployer key — NEVER SHARE OR COMMIT ────────────────
PRIVATE_KEY=0xYOUR_PRIVATE_KEY_HERE
⚠️ Replace 0xYOUR_PRIVATE_KEY_HERE with the actual private key you saved in Step 2B. Replace 0xYOUR_ADMIN_WALLET_ADDRESS with your wallet address from Step 2C.
C

Install dependencies (if not done yet)

npm install

This takes 1–2 minutes. Warnings about deprecated packages are normal — ignore them.

4
Deploy Smart Contracts to Polygon Amoy
This puts your contracts on the real blockchain — permanently and for free
A

Run the deploy command

In your Terminal (make sure you're in the blockchain-voting folder):

npm run deploy:amoy

This will take about 30–60 seconds. You should see:

Deploying to Polygon Amoy (testnet)
Deployer: 0xYOUR_ADDRESS
Balance: 0.5 MATIC

Deploying ElectionFactory...

✅ ElectionFactory deployed!
   Address: 0xABC123DEF456...
   Tx Hash: 0x789...
   Explorer: https://amoy.polygonscan.com/address/0xABC123...

📝 .env.local updated.
B

Verify it worked

1. Open your .env.local file — you'll see NEXT_PUBLIC_FACTORY_ADDRESS is now filled in automatically.
2. Copy that address and paste it into amoy.polygonscan.com — you should see the contract page.

✅ If you see the contract on PolygonScan, it's live on the blockchain forever. Even if you close your Mac right now, it's there.
C

If deployment fails

Error: "insufficient funds" → Your wallet doesn't have MATIC. Go back to the faucet (Step 2E).

Error: "could not detect network" → Check your NEXT_PUBLIC_RPC_URL in .env.local is exactly https://rpc-amoy.polygon.technology/

Error: "invalid private key" → Make sure your PRIVATE_KEY starts with 0x and is 66 characters long.

5
Push Code to GitHub
Create a repository and push your code — private key stays safe
A

Create a new GitHub repository

1. Go to github.com → click the "+" icon (top right) → "New repository"
2. Fill in:
   • Repository name: blockchain-voting
   • Description: Decentralised blockchain voting app
   • Visibility: Private (recommended) or Public
3. Do NOT tick "Add a README file" or "Add .gitignore" — the project already has these
4. Click "Create repository"

GitHub will show you a page with commands. Keep that page open.

B

Confirm .env.local is excluded

Run this to make absolutely sure your private key won't be uploaded:

cat .gitignore | grep env

You should see .env.local in the output. If you do — you're safe to continue.

C

Initialise Git and push

Run these commands one by one in your Terminal:

# Step 1: Initialise git in your project
git init

# Step 2: Add all files
git add .

# Step 3: Check what's being added (.env.local should NOT appear)
git status

# Step 4: Create the first commit
git commit -m "feat: ChainVote blockchain voting app"

# Step 5: Rename the branch to main
git branch -M main

# Step 6: Connect to your GitHub repo (replace YOUR_USERNAME)
git remote add origin https://github.com/YOUR_USERNAME/blockchain-voting.git

# Step 7: Push to GitHub
git push -u origin main

When prompted, enter your GitHub username and password. If GitHub asks for a token instead of password, go to: github.com → Settings → Developer settings → Personal access tokens → Generate new token (classic) → tick "repo" → copy the token and use it as your password.

D

Verify on GitHub

Go to github.com/YOUR_USERNAME/blockchain-voting — you should see all your files. Confirm that .env.local is NOT listed.

✅ Your code is now on GitHub. Your private key is safe (not uploaded).
6
Deploy Frontend to Vercel
Free hosting — your app gets a public URL anyone can access
A

Install Vercel CLI

npm install -g vercel

Verify it installed:

vercel --version
B

Login to Vercel

vercel login

It will ask how to log in — choose "Continue with GitHub". Your browser will open and ask you to authorise. Click Authorise.

You'll see ✅ Logged in in the terminal.

C

Deploy to Vercel

Make sure you're in the blockchain-voting folder, then run:

vercel --prod

Answer the questions like this:

? Set up and deploy? → Y (press Enter)
? Which scope? → select your username
? Link to existing project? → N
? What's your project's name? → blockchain-voting (press Enter)
? In which directory is your code located? → ./ (press Enter)
? Want to modify these settings? → N (press Enter)

Vercel will build and deploy. After ~2 minutes you'll see:

✅  Production: https://blockchain-voting-abc123.vercel.app

Copy this URL — this is your live app!

7
Set Environment Variables on Vercel
The app needs your contract address and network settings to work
A

Open Vercel Dashboard

Go to vercel.com/dashboard → click on your blockchain-voting project → click Settings (top tab) → click Environment Variables (left sidebar).

B

Add each variable

For each row below: type the name in the "Key" box, paste the value in "Value", make sure Production, Preview, Development are all ticked, then click Save.

Key (Name)Value
NEXT_PUBLIC_RPC_URLhttps://rpc-amoy.polygon.technology/
NEXT_PUBLIC_CHAIN_ID80002
NEXT_PUBLIC_CHAIN_HEX0x13882
NEXT_PUBLIC_CHAIN_NAMEPolygon Amoy
NEXT_PUBLIC_FACTORY_ADDRESSPaste from your .env.local (starts with 0x)
NEXT_PUBLIC_EXPLORER_BASE_URLhttps://amoy.polygonscan.com
NEXT_PUBLIC_ADMIN_ADDRESSYour ChainVote Admin wallet address
⚠️ Do NOT add PRIVATE_KEY to Vercel. That key is only needed for deploying contracts, never for the frontend.
C

Redeploy to apply the variables

Environment variables only apply after a fresh deployment. Run:

vercel --prod

Or in the Vercel dashboard → Deployments tab → click the three dots on the latest deployment → Redeploy.

8
Test the Live App End-to-End
Make sure everything works before sharing with students
A

Open your live URL

Go to your Vercel URL (e.g. https://blockchain-voting-abc123.vercel.app). You should see the ChainVote landing page with the amber design.

B

Connect admin wallet and create a test election

1. Make sure MetaMask is on Polygon Amoy and using your ChainVote Admin account
2. Click Admin in the nav → Connect MetaMask
3. You should see a green Admin badge next to your address
4. Create a test election with start = now, end = 1 hour from now
5. Approve the MetaMask popup (costs ~0.001 MATIC)
6. Activate the election

C

Register a voter and test voting

1. In MetaMask, create or switch to a second account (Account #2)
2. Copy its address
3. In the Admin Portal → Manage → Register Voters → paste the address → Register
4. Switch MetaMask to Account #2
5. Go to the election ballot page → select a candidate → Vote
6. Approve in MetaMask → vote recorded! ✅

D

Verify the transaction on PolygonScan

1. After voting, scroll down on the ballot page to the On-Chain Transaction Log
2. Click the transaction hash link → PolygonScan opens
3. You'll see the real blockchain transaction with timestamp, block, and gas used

✅ If you can see it on PolygonScan — that vote is permanent. It will be there forever even if you delete your app, close your laptop, or the Vercel site goes down.
9
How to View Transactions on PolygonScan
Anyone can verify the election results independently
A

View all elections (factory contract)

Go to:

https://amoy.polygonscan.com/address/YOUR_FACTORY_ADDRESS

You'll see every transaction — including every election that was created.

B

View votes for a specific election

Copy the election contract address from the ballot page (Contract Details section) and paste it into PolygonScan:

https://amoy.polygonscan.com/address/ELECTION_CONTRACT_ADDRESS

Click the Events tab to see every VoteCast event — one per vote.

C

What anyone can see vs what's private

Visible on PolygonScanPrivate / Not shown in UI
✅ A vote happened🔒 Which candidate was chosen
✅ Exact timestamp🔒 (candidate ID is encoded in tx data)
✅ Block number 
✅ Transaction hash 
✅ Total vote count 
✅ Gas fee paid (tiny) 
ℹ️ The voter's wallet address IS technically visible on PolygonScan in the raw transaction data — this is how public blockchains work. Your app's UI never displays who voted for whom. For a classroom setting this level of privacy is appropriate and expected.
